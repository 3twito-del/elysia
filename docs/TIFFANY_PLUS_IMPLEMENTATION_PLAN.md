# תוכנית יישום לרמת Tiffany ומעלה

מטרת המסמך היא להפוך את הדיון על “רמת Tiffany ומעלה” לתוכנית עבודה ישימה בקוד. הדגש הוא לא להוסיף עוד שכבת עיצוב כללית, אלא לחזק את נקודות ההחלטה במסחר: דף מוצר, תהליך תשלום, חיפוש, קטגוריות, כרטיסי מוצר ואמון.

## סטטוס יישום

- Product Page Pass: יושם ונבדק.
- Checkout Confidence Pass: יושם ונבדק.
- Search / Category Polish: יושם ונבדק.
- Product Cards Luxury Pass: יושם דרך decision facts, מצב זמינות/מבצע יציב, ושורת אמון קומפקטית בכרטיסי מוצר.
- Trust Layer: יושם דרך שכבת אמון גלובלית בפוטר וקישורי שירות/מידות/אחריות/מסירה.
- Visual QA Mobile First: כוסה במסמך QA ייעודי ובבדיקות browser smoke.
- Guardrails: הורחבו בבדיקות סטטיות שמכסות כרטיסי מוצר, trust layer, active states ו-QA.

## עקרונות

- מובייל קודם: כל שינוי נבדק קודם ב־390px ואז בדסקטופ.
- אין חזרה לטורקיז הישן, gradients דקורטיביים, או כרטיסים בתוך כרטיסים.
- כל רכיב אינטראקטיבי חייב להציג מצב פעיל ברור.
- טקסט לא נחתך, לא חופף, ולא מסתיר פעולות קנייה.
- כל שינוי עיצובי מקבל guardrail בבדיקה סטטית או browser smoke.

## 1. Product Page Pass

מטרה: להפוך דף מוצר לעמוד החלטת קנייה ברמת פרימיום.

שינויים שניתן ליישם:

- להדק את גלריית התמונות במובייל ובדסקטופ.
- לוודא שמחיר, זמינות, חומר, מידה ומשלוח מופיעים לפני שהמשתמש צריך לגלול הרבה.
- לחדד את ה־CTA הראשי ואת מצב הבחירה של מידה/וריאציה.
- להצמיד “שירות אישי על הפריט” לרגע ההחלטה.
- לשלב מדריך מידות, החזרות ואחריות באזור הקנייה במקום להסתמך על קישורי צד בלבד.
- להוסיף בדיקות contract ל־PDP שמונעות נסיגה במבנה, CTA, trust facts וגלריה.

אימותים:

- browser smoke על דף מוצר במובייל ובדסקטופ.
- `pnpm lint`
- `pnpm typecheck`
- `pnpm test`
- `pnpm build`

## 2. Checkout Confidence Pass

מטרה: להוריד חיכוך ופחד לפני תשלום.

שינויים שניתן ליישם:

- לשפר את סיכום ההזמנה במובייל.
- להבליט אמון בתשלום, משלוח, החלפה והחזרות.
- לשפר הודעות שגיאה בטפסים כך שהמשתמש יודע מה לתקן.
- לחזק recovery לעגלה ריקה או checkout לא תקין.
- לוודא שמידע שירותי לא מרגיש כמו הבטחה משפטית חדשה, אלא הצגה ברורה של מה שכבר קיים באתר.

אימותים:

- browser smoke על checkout במובייל.
- בדיקות validation קיימות + הרחבה לפי הצורך.
- `pnpm build`

## 3. Search / Category Polish

מטרה: להפוך מציאת מוצר לפעולה ברורה, לא מערכתית.

שינויים שניתן ליישם:

- לוודא שהבדל בין טבלה, שורות, סינון ומיון ברור בכל viewport.
- לצמצם עומס טקסטואלי באזור תוצאות.
- לשפר active filters במובייל.
- לוודא שכל כפתור מצב מציג active state ברור.
- להרחיב בדיקות שמונעות חזרה למצב שבו שני כפתורים נראים זהים.

אימותים:

- browser smoke על `/search`, `/search?view=list`, ועמוד קטגוריה.
- contract tests לתצוגות, filters ו־active state.

## 4. Product Cards Luxury Pass

מטרה: כרטיסי המוצר צריכים להרגיש כמו חלון ראווה, לא רשומת קטלוג.

שינויים שניתן ליישם:

- היררכיה נקייה יותר לשם מוצר, מחיר, חומר וזמינות.
- יחס תמונה עקבי יותר.
- hover/focus עדין ללא קפיצות layout.
- פעולות wishlist/quick actions בלי עומס.
- טיפול מדויק במצבי unavailable/sale.

אימותים:

- בדיקות layout stability.
- browser smoke בקטגוריה, חיפוש ו־wishlist.

## 5. Trust Layer

מטרה: להפוך אמון למערכת עקבית בכל האתר.

שינויים שניתן ליישם:

- בלוקים קצרים על משלוח, החזרות, אחריות ושירות.
- מיקום נכון ב־PDP, checkout, footer ו־service.
- ניסוח קצר, עובדתי ולא שיווקי מדי.
- קישורים ברורים לשירות אישי, מדריך מידות והחזרות.

אימותים:

- contract tests שמוודאים שה־trust layer מופיע בנקודות ההחלטה.
- בדיקת copy כדי שלא ייווצרו הבטחות לא מאומתות.

## 6. Visual QA Mobile First

מטרה: לזהות חפיפות, חיתוכי טקסט, states לא ברורים ובעיות real-device לפני פרוד.

מסכים לבדיקה:

- Home
- Category
- Search
- Product
- Checkout
- Wishlist
- Service

בדיקות נדרשות:

- viewport מובייל 390x844.
- viewport דסקטופ 1440x900.
- console errors.
- צילום מסך לפני/אחרי לשינויים משמעותיים.
- בדיקת כפתורים במצבים פעילים/לא פעילים.

## 7. Guardrails

מטרה: למנוע נסיגה לרמת UI פחות יוקרתית.

בדיקות שניתן להוסיף או להרחיב:

- אין טורקיז ישן או legacy aqua.
- אין gradients דקורטיביים.
- אין כרטיסים בתוך כרטיסים.
- אין כפתורי מצב שנראים זהים.
- אין טקסט חתוך במובייל.
- אין letter-spacing חריג בטקסט ציבורי.
- אין icons חוזרים ברשימות סטטיות שונות ללא סיבה.

## סדר ביצוע מומלץ

1. Product Page Pass
2. Checkout Confidence Pass
3. Search / Category Polish
4. Product Cards Luxury Pass
5. Trust Layer
6. Visual QA Mobile First

השלב הראשון המומלץ הוא Product Page Pass, כי שם הלקוח מחליט אם לקנות. זה המקום שבו ההבדל בין אתר יפה לבין luxury commerce מורגש בצורה הכי ישירה.
