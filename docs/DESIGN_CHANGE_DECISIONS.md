# Design Change Decisions

## Purpose

This file records only approved design changes for Aphrodite.

The working flow is:

1. The user proposes a design change.
2. Codex responds with agreement or concrete concerns.
3. The user makes the final decision.
4. Only approved changes are added here.
5. When this file is ready for execution, Codex implements the approved changes.
6. After implementation, Codex uploads all changes through the agreed workflow.
7. Every implemented change is deleted from the approval log so the document is
   ready for new decisions.

Rejected or undecided ideas are intentionally not recorded in this file.
